# Contributing to ADD 1.0

Thank you for your interest in contributing to Agent-Driven Development (ADD) 1.0!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Search existing issues before creating a new one
- Provide clear, detailed information
- Include examples when applicable

### Improving Documentation

- Fix typos, clarify explanations, add examples
- Follow ADD principles when documenting
- Keep documentation concise (follow our optimization approach)
- Update relevant sections only

### Proposing Changes

1. **Fork** the repository
2. **Create a branch** following our naming convention:
   - `feat/description` for new features
   - `fix/description` for bug fixes
   - `docs/description` for documentation
   - `chore/description` for maintenance
3. **Follow ADD 1.0** methodology in your changes
4. **Commit** using Conventional Commits format
5. **Test** your changes
6. **Submit** a Pull Request

## Commit Message Format

Use Conventional Commits:

```
<type>: <description>

[optional body]
```

Types: `feat`, `fix`, `docs`, `chore`, `test`, `refactor`, `perf`, `style`, `ci`, `build`

Examples:
```
feat: add exit criteria for MARKET phase
fix: correct versioning example
docs: update IDE configuration guide
chore: update dependencies
```

## Pull Request Guidelines

- Fill out the PR template completely
- Reference related issues
- Keep changes focused and atomic
- Ensure all checks pass
- Update documentation if needed
- Follow the code review feedback

## Standards

- **Concise**: Remove unnecessary content
- **Clear**: Use simple, direct language
- **Consistent**: Follow existing patterns
- **Standards-First**: Prefer established standards (semver, Conventional Commits, etc.)
- **No duplication**: Don't repeat information

## Code of Conduct

This project follows the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.

## Questions?

Open a GitHub Discussion or issue for questions about contributing.

---

**Remember**: When in doubt, follow ADD 1.0 principles in your contributions!
